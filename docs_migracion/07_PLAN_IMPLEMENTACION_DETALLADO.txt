1. DISEÑO ARQUITECTÓNICO Y ESTRUCTURA DE DATOS
================================================================================
La arquitectura propuesta separa la aplicación en tres capas lógicas para 
garantizar la mantenibilidad, seguridad y rendimiento del sistema.

1.1 CAPA DE PRESENTACIÓN (Frontend React SPA)
  - Tecnología: React 18+ con TypeScript, empaquetado por Vite.
  - Gestión de Formularios: React Hook Form para un manejo óptimo de los renders.
  - Validación del Cliente: Esquemas Zod para validación tipada y en tiempo real.
  - Comunicación: Axios + TanStack Query (React Query) para llamadas y caché de API.
  - Sistema de Diseño: CSS Vanilla con CSS Modules y Variables CSS para un look
    y feel premium, responsivo y con soporte de modo oscuro.

1.2 CAPA DE LÓGICA DE NEGOCIO (Backend API REST)
  - Tecnología: Node.js con Express y TypeScript.
  - Seguridad: Autenticación por Tokens JWT y control de acceso por roles.
  - Sanitización de Entradas: Middleware de validación que intercepta cuerpos de 
    request y los valida contra esquemas Zod en el servidor.
  - Logger: Morgan para auditoría de llamadas HTTP.
  - Manejo de Errores: Middleware globalizador de excepciones HTTP.

1.3 CAPA DE PERSISTENCIA (Base de Datos SQL Server)
  - Esquema Normalizado: Separación de la tabla monolítica tbl_DGenerales en 
    4 tablas 1:1 especializadas y 5 tablas 1:N asociadas.
  - Encapsulamiento SQL: Operaciones complejas aisladas en Stored Procedures.
  - Integridad Referencial: Llaves foráneas con borrado en cascada (ON DELETE CASCADE)
    para evitar inconsistencias o datos huérfanos.

1.4 MODELO DE DATOS NORMALIZADO
  - tbl_candidatos: Datos de identidad, teléfonos y residencia.
  - tbl_candidatos_salud_deportes: Enfermedades, alergias y deportes (1:1).
  - tbl_candidatos_puesto: Puesto solicitado, salarios y resumen laboral (1:1).
  - tbl_candidatos_emergencia: Nombre, parentesco y celular de emergencia (1:1).
  - tbl_datos_familiares: Relación 1:N de familiares del candidato.
  - tbl_datos_academicos: Relación 1:N de títulos e instituciones.
  - tbl_experiencia_profesional: Relación 1:N de historial laboral.
  - tbl_referencias: Relación 1:N de referencias personales/laborales.
  - tbl_idiomas: Catálogo maestro de idiomas.
  - tbl_candidato_idiomas: Relación N:M de idiomas dominados con niveles.
