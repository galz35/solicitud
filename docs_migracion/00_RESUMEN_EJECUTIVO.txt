1. INTRODUCCIÓN Y ESTRUCTURA DE LA DOCUMENTACIÓN
================================================================================
Este documento sirve como guía principal e índice del plan de migración tecnológica
para el sistema "Solicitud de Empleo". El objetivo es reemplazar el sistema legacy
basado en ASP Clásico por una arquitectura moderna, segura, escalable y con una
experiencia de usuario óptima utilizando React en el frontend y una API REST en el backend.

2. ESTRATEGIA TECNOLÓGICA PROPUESTA (STACK MODERNO)
================================================================================
Para la reescritura total del sistema, se propone el siguiente stack técnico:

  - FRONTEND: React.js (Vite o Next.js) + TypeScript
    * Tailwind CSS o CSS Modules para un diseño premium, responsive y fluido.
    * React Hook Form + Zod para validaciones robustas del lado del cliente.
    * Axios + React Query (TanStack Query) para el manejo de estado de datos y caché.
    * Lucid React / Heroicons para iconografía moderna.

  - BACKEND: API REST (Node.js/Express, ASP.NET Core o Python FastAPI)
    * Arquitectura limpia en capas (Controladores, Servicios, Repositorios).
    * Seguridad: Autenticación basada en JWT, encriptación de datos sensibles.
    * Validación de inputs redundante en servidor.

  - BASE DE DATOS: SQL Server (Existente pero modernizada)
    * Normalización de tablas (tbl_DGenerales dividida lógicamente).
    * Implementación de Stored Procedures para encapsular lógica de negocio SQL.
    * Migraciones versionadas (DbUp o Entity Framework Migrations).

3. PRINCIPALES BENEFICIOS DE LA MIGRACIÓN
================================================================================
  - SEGURIDAD: Eliminación completa de SQL Injection, credenciales quemadas en código
    y ataques XSS. Introducción de autenticación segura.
  - RESPONSIVIDAD (MOBILE FIRST): El candidato podrá llenar su solicitud cómodamente
    desde cualquier teléfono inteligente o tablet.
  - MANTENIBILIDAD: De un archivo monolítico de más de 2,000 líneas a componentes
    modulares menores a 200 líneas fáciles de leer, testear y extender.
  - REDUCCIÓN DE ERRORES: Validación en tiempo real y paso a paso en el formulario
    evitando que el usuario envíe datos erróneos o incompletos.
  - AUDITORÍA Y CONTROL: Registro de modificaciones (auditoría en BD) para saber qué
    y cuándo se actualizó la información de un candidato.
